如何解决RTSS和OBS的游戏捕获之间的共存？

发表于 2024-02-25 分类于 PC Waline：

.

（本文只有在需要用管理员权限打开的时候才会标注用管理员权限打开，其余时候管不管理员打开都无所谓）
传统办法：

1.RTSS 和 OBS游戏捕获窗口舍其一
（本身obs在官网也有说明，OSD类应用尤其是RTSS可能存在和OBS捕获之间的冲突。）

2.关闭游戏和obs，在rtss内的设置（setup）勾选“Use Microsoft Detours API hooking”后重新启动rtss，再重新打开游戏和obs，用obs捕获游戏窗口此时正常（但在obs游戏捕获窗口内没有OSD信息，也就是说录下来的视频内不会有osd监控信息）

但有完美的解决办法：
3.如果打开obs捕获游戏窗口时（无论是否用管理员权限打开OBS），就已经打开了游戏和RTSS，那么不关机重启的情况下，哪怕已经关闭了小飞机和rtss，重启了游戏和OBS程序，也会无法捕获到单独的这个进程游戏窗口（也就是说只是捕获不到这个游戏而已，其他的窗口没影响）

解决这个问题，也同时是让游戏捕获捕获到这个窗口+OSD监控信息的办法：
关闭小飞机和rtss和obs，重启游戏后，再用管理员权限打开obs，然后发现已经可以捕获到游戏画面，这时候再打开小飞机，捕获的游戏画面依旧没问题，但osd的监控信息也可以录下来了。
而且，这么做之后，只要不重启游戏，那么中间无论怎么关闭重启OBS和小飞机/RTSS，捕获的游戏画面依旧没问题，OSD的监控信息也都能被录进视频。
